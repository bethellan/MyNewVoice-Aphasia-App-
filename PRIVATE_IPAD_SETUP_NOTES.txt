MyNewVoice private device setup notes - v6

This public GitHub app can remain generic. Real photos and recorded voices can now be added on each device through Settings > Photo / Voice Setup.

What is stored locally:
- Top menu category images
- Phrase-row images
- Optional zoom images shown for 4 seconds
- Recorded voice clips for phrases

Storage behaviour:
- Private photos and voices are saved in the browser on the current device using IndexedDB.
- They are not uploaded to GitHub.
- They should reappear when the app is reopened on the same device/browser.
- They can be lost if the browser clears site data or storage is evicted.

Recommended workflow:
1. Open the app from the GitHub Pages HTTPS address or installed PWA.
2. Tap Settings (gear).
3. Tap Photo / Voice Setup.
4. Tap Protect local storage.
5. Choose photos and record voices.
6. Use Export private backup after setup. Keep that backup file somewhere safe.
7. Use Import private backup to copy the same private setup onto another device.

Voice recording:
- The browser will ask for microphone permission.
- Recording usually requires the app to be opened from an HTTPS address, such as GitHub Pages.
- If a recorded voice exists for a phrase, tapping the phrase plays the recorded voice.
- If no recorded voice exists, the app uses normal text-to-speech.

Image priority:
1. Local private image saved on the device
2. Image file in the GitHub folder
3. Fallback icon

Speech priority:
1. Local recorded voice saved on the device
2. Browser text-to-speech
